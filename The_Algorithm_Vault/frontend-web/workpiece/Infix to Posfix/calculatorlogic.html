    บทคัดย่อ (Abstract)

    บทความนี้นำเสนอการวิเคราะห์เชิงวิจัยของกระบวนการแปลงนิพจน์ Infix เป็น Postfix และการประเมินผล Postfix ผ่านโมเดล Stack-based Execution System ที่ถูกนำไปใช้ใน Math Engine Lab นี้

    การวิเคราะห์ครอบคลุม: Formal Algorithm Definition, Operator Precedence Model, Worst / Average / Amortized Complexity, Memory Complexity, Correctness Proof, และ Execution Trace Modeling

    1. Formal Problem Definition

    ให้ Infix Expression E ที่มีความยาว n tokens ต้องการแปลงเป็น Postfix Expression P โดยรักษาลำดับความสำคัญของ Operator 

    
Operators:
+  -  precedence = 1
*  /  %  precedence = 2
(  precedence = 0

    กำหนด precedence function: 

    precedence(op)

    2. Shunting Yard Algorithm

    
For each token t in infix:
    if operand → output
    if '(' → push to stack
    if ')' → pop until '('
    if operator:
        while precedence(top) >= precedence(t)
            pop to output
        push t

    เมื่อจบ traversal: 

    
while stack not empty:
    pop to output

    3. Time Complexity Analysis

    
Let n = number of tokens
Each token:
    pushed at most once
    popped at most once

    
Worst Case  : O(n)
Average Case: O(n)
Amortized   : O(n)

    แม้จะมี while-loop ซ้อน แต่รวมการ pop ทั้งหมดไม่เกิน n ครั้ง จึงเป็น Linear Time

    4. Memory Complexity

    
Operator Stack ≤ n
Postfix Output ≤ n
Operand Stack ≤ n

    
Total Space = O(n)

    ระบบนี้ใช้ stack สองชุด: Operator Stack และ Operand Stack ซึ่งในกรณีเลวร้ายสุดอาจโตถึง O(n)

    5. Postfix Evaluation Model

    
For each token t in postfix:
    if operand → push
    if operator:
        b = pop()
        a = pop()
        push(a op b)

    Complexity: 

    
Time  = O(n)
Space = O(n)

    การประเมินผลเป็น deterministic stack machine เทียบเท่า Reverse Polish Notation VM

    6. Correctness Proof (Sketch)

    พิสูจน์โดย Induction บนจำนวน token: 

    
Base Case:
1 operand → correct

Inductive Step:
Assume k tokens correct
Token k+1 preserves precedence order

    การใช้ stack ทำให้: Higher precedence operators ถูก evaluate ก่อนเสมอ

    7. Mapping to Implementation

    
const precedence = {
 '+':1, '-':1,
 '*':2, '/':2, '%':2,
 '(':0

while (opStack.length > 0 &&
       precedence[opStack[top]] >= precedence[char]) {
    postfixStr += opStack.pop();
}

    8. System-Level Insight

    • Algorithm เป็น Linear Stack Machine
    • ไม่มี Backtracking
    • Deterministic Execution
    • Space usage predictable

    โมเดลนี้สามารถขยายไปสู่: Expression Compiler, Bytecode Generator, หรือ Mathematical Interpreter

    9. บทสรุปเชิงวิจัย

    Infix to Postfix Transformation เป็นปัญหาคลาสสิกที่แสดงให้เห็นพลังของ Stack ในการจัดการ Operator Precedence

    Algorithm นี้มี Time Complexity O(n) และ Memory Complexity O(n) พร้อม Proof-based correctness
