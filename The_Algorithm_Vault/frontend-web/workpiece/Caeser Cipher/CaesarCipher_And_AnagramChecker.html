    Caesar Cipher & Anagram Engine: Formal Analysis

    โมเดลเชิงคณิตศาสตร์ของการเข้ารหัสแบบ Substitution Cipher และการตรวจสอบความเป็น Permutation

    1. โมเดลทางคณิตศาสตร์ของ Caesar Cipher

    Caesar Cipher เป็นการเข้ารหัสแบบ Shift Cipher ซึ่งนิยามได้ในเชิงฟังก์ชัน:

    E(x) = (x + k) mod 26

    โดยที่:

    • x = ตำแหน่งของตัวอักษรใน Alphabet (0-25)
    • k = ค่า shift
    • mod 26 = การวนรอบตัวอักษร

    2. ฟังก์ชันถอดรหัส (Inverse Function)

    D(x) = (x - k) mod 26

    ในระบบของคุณ Decryption ถูกเขียนในรูป:

    26 - (k mod 26)
    ซึ่งเป็น equivalent form ของการลบใน modular arithmetic

    3. Modular Arithmetic Formalization

    ระบบตัวอักษรสามารถมองเป็น Group:

    (Z₍26₎, +)

    Caesar Cipher คือการทำ translation ใน cyclic group

    4. Time Complexity Analysis

    ให้ n = ความยาวข้อความ

    T(n) = n
    • Worst-case = O(n)
    • Average-case = O(n)
    เนื่องจากต้องประมวลผลทุกตัวอักษร

    5. Memory Complexity

    ใช้:
    • String split → O(n)
    • Map result → O(n)
    M(n) = O(n)

    6. Security Weakness Analysis

    Caesar Cipher มี Key space เพียง:
    |K| = 26
    ดังนั้น brute-force complexity คือ:
    O(26n) ≈ O(n)
    ซึ่งถือว่าไม่ปลอดภัยในเชิง Cryptographic Standard

    7. Frequency Analysis Vulnerability

    เนื่องจากเป็น Monoalphabetic Substitution Distribution ของตัวอักษรถูกเลื่อนเท่านั้น ดังนั้น Frequency Profile ยังคงรูปแบบเดิม ทำให้สามารถใช้ Statistical Attack ได้

    8. Anagram Checker as Permutation Test

    ให้ s₁ และ s₂ เป็น string สองชุด Anagram คือ:
    sort(s₁) = sort(s₂)
    ซึ่งหมายถึง:
    Multiset(s₁) = Multiset(s₂)

    9. Time Complexity ของ Anagram

    ขั้นตอน:
    • Split → O(n)
    • Sort → O(n log n)
    • Compare → O(n)
    ดังนั้น:
    T(n) = O(n log n)

    10. Alternative Optimization (Hash Counting)

    สามารถลด complexity เหลือ:
    O(n)
    โดยใช้ frequency array ขนาด 26
