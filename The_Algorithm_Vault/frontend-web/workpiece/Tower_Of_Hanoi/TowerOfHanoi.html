    Tower of Hanoi : Deep Theoretical & Computational Analysis

    Recursive Structure, Exponential Growth, Memory Model และ State Space Complexity

    Abstract

    บทความนี้วิเคราะห์อัลกอริทึม Tower of Hanoi ทั้งในเชิงคณิตศาสตร์และเชิงคำนวณ ครอบคลุม Recurrence Relation, Closed-form solution, Exponential Time Complexity, Stack Memory Behavior, และ State Space Explosion พร้อมเชื่อมโยงกับ implementation ใน Interactive Lab

    1. Formal Recursive Model

    T(n) = 2T(n-1) + 1

    โดยมี base case:

    T(1) = 1

    Closed-form solution:

    T(n) = 2^n - 1

    2. Mathematical Induction Proof

    Base Case:

    2¹ - 1 = 1 ✓

    Inductive Step:

    T(k+1) = 2(2^k -1)+1 = 2^(k+1)-1

    3. Time Complexity Analysis

    Worst-case

    Θ(2^n)

    Tower of Hanoi ไม่มี average-case หรือ best-case เพราะทุกกรณีต้องทำครบทุกขั้นตอน

    4. Space Complexity (Call Stack Analysis)

    • Maximum recursion depth = n
    • Each stack frame → O(1)
    Space = O(n)

    แม้ time จะเป็น exponential แต่ memory โตแบบ linear

    5. State Space Explosion

    แต่ละจานสามารถอยู่ได้ 3 เสา

    |State Space| ≤ 3^n

    แต่ optimal path มีความยาวเพียง:

    2^n - 1

    6. Recursion Tree Interpretation

    โครงสร้างการเรียกเป็น Binary Recursion Tree

    Nodes ≈ 2^n

    ซึ่งเป็นเหตุผลของ exponential growth

    Conclusion

    Tower of Hanoi เป็นตัวอย่างคลาสสิกของ Exponential Recursive Algorithm ที่มี Closed-form solution ชัดเจน และใช้สอนเรื่อง Big-O, Induction, Recursion Tree, และ Stack Memory ได้อย่างมีประสิทธิภาพ

    HanoiSolver.js : Recursive Engine Deconstruction

    1. Core Recursive Function

    moveStack(n, target)

    Algorithm:

    1. ถ้า n = 0 → return
    2. ย้าย n-1 ไป auxiliary
    3. ย้ายจาน n ไป target
    4. ย้าย n-1 ทับตาม

    2. Arbitrary-State Solver

    เวอร์ชันใน Lab รองรับสถานะเริ่มต้นใดๆ

    • ค้นหาตำแหน่งจาน n
    • คำนวณ auxiliary pole
    • แก้แบบ recursive จากสถานะปัจจุบัน

    ทำให้สามารถ Resume จาก state กลางเกมได้

    3. Behavioral Insight

    AI Solve ทำงานตามลำดับ optimal และจะทำครบ 2^n - 1 moves เสมอ

    4. Memory vs Time Contrast

    Metric Growth
    Time Θ(2^n)
    Space O(n)

    Key Insight สำหรับผู้เรียนขั้นสูง

    • Exponential growth ไม่ได้หมายถึง memory exponential
    • Recursion depth ≠ total calls
    • Binary recursion tree อธิบาย complexity ได้ชัด
    • Closed-form ช่วยยืนยัน recurrence correctness
