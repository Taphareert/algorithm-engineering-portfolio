1. Linked List: Dynamic Memory Architecture

1.1 โครงสร้างเชิงทฤษฎี

Linked List เป็นโครงสร้างข้อมูลแบบ Node-based Dynamic Structure โดยแต่ละโหนดประกอบด้วย: 


Node = { data, next }

ในเชิงหน่วยความจำ:


Memory = Σ (Data_i + Pointer_i)

โครงสร้างนี้ไม่มี contiguous allocation และใช้ heap allocation ทีละ node เชื่อมโยงกันด้วย pointer

1.2 Time Complexity


Insert Head      : O(1)
Delete Head      : O(1)
Search           : O(n)
Delete by value  : O(n)

1.3 Memory Complexity


Space(n) = n × (Data + Pointer)
Memory Complexity = O(n)

บนระบบ 64-bit pointer มีขนาด 8 bytes หาก data เป็น int (4 bytes) จะมี overhead ต่อ node สูงกว่า array อย่างมีนัยสำคัญ

1.4 Cache Locality Impact

Linked List มี poor spatial locality เนื่องจาก pointer ทำให้เกิด cache miss และ pointer chasing cost สูง

1.5 Mapping กับ Implementation ในโปรเจคนี้


let nodes = [10, 25, 40, 55];

แม้ใช้ Array เป็น storage จริง แต่ algorithm ถูกจำลองแบบ pointer traversal ผ่าน state-driven animation

2. Stack: Static Contiguous Model

2.1 ทฤษฎีพื้นฐาน


Push(x) → Top = Top + 1
Pop()   → Top = Top - 1

2.2 Java Implementation Model


public void push(int x) {
    if(top >= size-1) return;
    stack[++top] = x;
}

2.3 Time Complexity


Push : O(1)
Pop  : O(1)
Peek : O(1)

2.4 Memory Complexity


Space(n) = O(N_max)

Stack แบบ static array จะจองหน่วยความจำเต็มตามขนาดสูงสุด แม้ใช้งานจริงจะไม่เต็ม

2.5 Cache Efficiency

Stack แบบ array มี excellent spatial locality เนื่องจากข้อมูลเรียงต่อเนื่องในหน่วยความจำ

3. Comparative Structural Analysis


LinkedList : Dynamic Allocation (Heap per node)
Stack      : Static Contiguous Allocation


LinkedList Memory = O(n)
Stack Memory      = O(N_max)

แม้บาง operation จะมี Big-O เท่ากัน แต่ constant factor และ cache behavior แตกต่างกันอย่างมาก

4. Formal Memory Proof


LinkedList:
S(n) = n(c_d + c_p)
⇒ Θ(n)

Stack:
S(n) = N_max × c_d
⇒ O(N_max)

Linked List มี pointer overhead ในขณะที่ Stack มี memory waste หากใช้งานไม่เต็ม capacity

5. Advanced System-Level Insight

Linked List เหมาะเมื่อ:

  • ขนาดข้อมูลไม่แน่นอน
  • มี insertion/deletion บ่อย
  • ไม่ต้องการ random access

Stack เหมาะเมื่อ:

  • ต้องการ LIFO execution
  • ต้องการ performance สูง
  • ต้องการ cache efficiency

ในระดับระบบจริง Stack แบบ Array มักเร็วกว่า Linked List แม้ Big-O จะเหมือนกันในบางกรณี เนื่องจาก pointer dereference cost และ cache miss penalty
